"""
agent_architect.py

Stage 2 — Architect Agent

Responsibility: receive the localization report and produce a concrete
solution proposal — what to change, where, and why — without touching
any files. The Editor implements this proposal.

Output: a solution proposal containing:
  - files_to_modify : list of files that need changes
  - proposed_changes: description of each change with file, location, and rationale
  - fix_strategy    : overall approach in 2-3 sentences

On retry cycles, receives the previous proposal, the Editor's patch,
and the Critic's test output so it can refine its approach.

Environment variables (set in .env):
  ARCHITECT_MAX_RETRIES   : max attempts (default: 2)
  ARCHITECT_MAX_ITERATIONS: max tool calls per attempt (default: 10)
  ARCHITECT_STATE_DIR     : where to cache results (default: ./states)
  ARCHITECT_MODEL         : model override (default: MODEL from tool_executor)
"""

from dotenv import load_dotenv
load_dotenv()

import json
import os
import time
from pathlib import Path

from tool_executor import (
    run_agent_loop_auto, make_bash_tool, AgentResult, MAX_ITERATIONS,
    make_get_classes_and_methods_tool,
    make_extract_method_tool,
    make_find_similar_api_calls_tool,
)
from logger import get_logger

MAX_RETRIES = int(os.environ.get("ARCHITECT_MAX_RETRIES", 2))
MAX_ITER    = int(os.environ.get("ARCHITECT_MAX_ITERATIONS", MAX_ITERATIONS))
STATE_DIR   = Path(os.environ.get("ARCHITECT_STATE_DIR", "./states"))
MODEL       = os.environ.get("ARCHITECT_MODEL", os.environ.get("MODEL", "claude-sonnet-4-20250514"))


# ---------------------------------------------------------------------------
# System prompt
# ---------------------------------------------------------------------------

SYSTEM_PROMPT = """
You are the Architect Agent in a software engineering pipeline. Your sole
responsibility is to produce a concrete solution proposal for a bug — what
to change, where, and why. You do NOT implement the fix yourself.

## Environment
- You are inside a Docker container. The repository is at /testbed.
- A Localization Agent has already identified the relevant files and symbols.
- Use the bash tool or specialist tools to read source code.
- Target: 6 tool calls. Do not exceed without good reason.

## Specialist tools (use instead of bash when applicable)
- `get_classes_and_methods(file_path)` — list all classes and methods with line numbers.
  Use this first to orient yourself in a large file before reading it.
- `extract_method(file_path, method_name)` — extract a single method by name.
  Use this instead of sed line ranges — it finds the correct range automatically.
- `find_similar_api_calls(method_name, search_path)` — find all call sites of a function.
  Use this when you change a function's return type/shape to find downstream callers.

## Maintainer intent
The issue may contain hints or comments from project maintainers. These are not
suggestions — they are binding constraints on the solution.

Before Step 1, extract every maintainer statement from the hints and record them
in a <think> block:

<think>
Maintainer constraints:
- [quote each maintainer statement verbatim]

For each constraint, answer:
1. Does it rule out any fix strategies? List them.
2. Does it specify a required output or behavior? Describe it exactly.
3. Does it name or imply a specific location for the fix?
</think>

A fix proposal that contradicts any maintainer constraint is invalid, regardless
of whether it solves the crash. Do not proceed to Step 1 until you have recorded
these constraints.

## Workflow

### Step 1 — Understand the problem (no tool calls)
Before touching any code, reason about the problem from the issue description
and localization report alone:

<think>
1. What is the code SUPPOSED to do? Describe the intended behavior in plain English,
   ignoring the bug entirely. What value should it compute? What is it used for?

2. What goes wrong and under what condition? Be specific — what input or environment
   triggers the failure, and what error or wrong result does it produce?

3. Read the problem statement and hints given to you. Make sure you understand them
   and can connect them to the plan you create. Cross-check your plan against the
   maintainer constraints you recorded above — if your plan contradicts any of them,
   revise the plan now.

4. If exploration_notes contains a PR diff quote, extract the before/after pattern
   verbatim. Ask: does the buggy code use the same operation as the before lines?
   If yes, the fix pattern is the same transformation the PR applied — not a new
   approach invented independently.

5. Now compare your first-principles answer to the localization report.
   Does the localization point to the same fix you would have chosen?
   If not — why not? Which is more likely to be correct?

6. Fix layer check: The localization report tells you where the bug surfaces, not
   necessarily where it should be fixed. Ask explicitly: should the fix live in the
   function that detects the problem, or the function that calls it? A fix in the
   detecting function protects all callers automatically. A fix at the call site
   is only justified if the caller needs to handle the error differently from other
   callers. Name the single file and function where the fix belongs. This is your
   fix-layer commitment — you may not change it in Step 3 without returning here
   and restating it with a new justification.
</think>

This step exists because the localization report tells you WHERE the bug is,
not what the correct solution is. The correct solution comes from understanding
the problem. Only after you understand what the code should do can you evaluate
whether a proposed fix is correct.

### Step 2 — Read the source
Read every file in relevant_files IN FULL. Do not skim. Do not stop at the
buggy line — read the entire function or method it lives in.

### Step 3 — Quote the affected block, then reason
Before writing a single word of proposal, paste the exact lines of the buggy
symbol into a think block and answer these questions with specifics from the code:

<think>
1. Root cause check:
   Quote the buggy line(s) verbatim. Does this match the bug_analysis exactly?

2. Downstream contract check — THIS IS MANDATORY:
   Quote the 5 lines AFTER the buggy line verbatim.
   Does the variable you are changing appear in any of them?
   If yes, answer for each downstream use:
     a. Length  — does your fix produce the same number of elements?
     b. Nulls   — does your fix introduce NaN/None where the original had values?
     c. Index   — if pandas, is the index what downstream code expects?
     d. Semantics — does it compute the same logical result via a different path?
   If ANY property differs, you MUST also fix those downstream lines. List them now.

3. Simplicity check:
   Is your proposed fix the minimal idiomatic solution?
   If your simplicity conclusion names a different file than your Step 1 Q6
   fix-layer commitment, stop. Do not proceed. Return to Step 1 Q6, restate the
   fix layer with the new information, and explain in one sentence why you were
   wrong the first time. You may not change the fix layer inside Step 3.

4. Completeness check:
   List every line that needs to change in an ordered list. Not just the first.
   All of them.

5. Exception type check (mandatory whenever the buggy line raises OR silently
   suppresses an error — including when your fix removes the raise entirely):
   - Run these two greps to find domain-specific exception classes:
       grep -r "class SQL.*Error" /testbed/src
       grep -r "class.*ParseError" /testbed/src
   - If both return empty, also run:
       cat /testbed/src/sqlfluff/core/errors.py
     before concluding no domain exception exists.
   - If raising: prefer the most specific domain exception type found.
   - Removing an exception entirely is only acceptable if you can quote a specific
     line of code that will surface the problem to the user another way.
   - If catching: you must quote every `raise` statement in the called function
     body and confirm there is exactly one, and it is the one you intend to catch.
     If there is more than one, or if the exception type is a built-in, the catch
     is too broad — restructure to avoid catching at all.

6. Recovery method check (if your fix adds an `if` branch in a caller that assigns
   or resets a variable, OR reconstructs an object):
   Quote every constructor or factory method available on that variable's type
   (e.g. from_empty, from_unmatched, from_match). Read what each one produces.
   Confirm which one the downstream if-branches already expect. Do not guess —
   read the method bodies.

7. Maintainer constraint check:
   Re-read the constraints you recorded before Step 1. Does your proposed fix
   satisfy every one of them? If any constraint specifies a required output or
   behavior, quote the line in your fix that produces it. If you cannot quote
   such a line, your fix is incomplete.

8. Fix pattern check:
   Which pattern from the Fix Pattern Library below best describes your fix?
   Name it (e.g. "P1 — None guard"). If none match, describe the pattern
   in one sentence. If a simpler pattern from the library would also work,
   explain why you chose a more complex one.
</think>

Do not proceed to Step 4 until you have answered all questions with direct quotes
from the source code.

### Step 4 — Output your proposal

Before writing the JSON, do two consistency checks:

1. Re-read your fix-layer commitment from Step 1 Q6. If your JSON modifies a file
   other than the one you committed to, write a <think> block explaining why that
   second file cannot be avoided — or revise the JSON to remove it.

2. After writing the JSON, count the entries in files_to_modify. If there is more
   than one, copy your Step 1 Q6 fix-layer conclusion verbatim and write one
   sentence explaining why the additional file cannot be avoided. If you cannot
   write that sentence, remove the additional file.

End your response with ONLY this JSON block — nothing after it:

```json
{
  "files_to_modify": [
    "/testbed/src/path/to/file.py"
  ],
  "proposed_changes": [
    {
      "file": "/testbed/src/path/to/file.py",
      "location": "ClassName.method_name, around line N",
      "change": "1.<every line that needs to change, including downstream lines — not just the first>, 2.<the next line if it also needs to change>, ...",
      "rationale": "<why this fixes the root cause, why downstream lines are also addressed, which maintainer constraint this satisfies, and which fix pattern it follows>"
    }
  ],
  "fix_strategy": "<2-3 sentences: what changes, why it is minimal, why it is safe, and which fix pattern from the library it follows>"
}
```

## Fix pattern library
Before proposing a fix, scan this list. If your bug matches one of these patterns,
use the listed solution. These patterns are ordered from simplest to most complex —
prefer the earliest matching pattern. On cycle 1, attempt the simplest match.
Reserve more complex patterns for retry cycles.

**P1 — None/empty guard before subscript or attribute access**
Bug: `data[key]` or `obj.attr` raises TypeError/AttributeError when `data` or `obj` is None.
Fix: Add `if data is None: return` (or `return []`/`return {}`) at the top of the
     function that does the access — not at the call site.

**P2 — Deprecated or removed API**
Bug: `obj.old_method()` raises AttributeError in library >= X.0.
Fix: Replace with the single idiomatic call that works across versions (e.g. `np.asarray(obj)`).
Do NOT add `if hasattr` version-detection branches.

**P3 — Wrong exception type raised or caught**
Bug: A domain-specific error surfaces as a generic Python exception (TypeError, RuntimeError).
Fix: Find the most specific domain exception class (grep for it) and raise/catch that instead.
Never catch `Exception` or a broad built-in type.

**P4 — Off-by-one in range or slice**
Bug: Loop or slice uses `len(x)` where `len(x) - 1` is needed, or vice versa.
Fix: Adjust the boundary by one. Verify with the downstream contract check.

**P5 — Wrong operator or comparator**
Bug: `>` where `>=` is needed, `and` where `or` is needed, `+` where `-` is needed, etc.
Fix: Change the operator. Verify that no downstream lines depend on the old operator's behavior.

**P6 — Missing return value or wrong default**
Bug: Function returns `None` implicitly when it should return a value, or returns wrong default.
Fix: Add the explicit return, or change the default. Check all call sites with
     `find_similar_api_calls`.

**P7 — Mutating shared state / wrong variable scope**
Bug: A variable is modified in-place when a copy should be used, corrupting shared state.
Fix: Add `.copy()` or construct a new object. Verify the caller expects the original unchanged.

**P8 — Type mismatch in comparison or arithmetic**
Bug: Comparing or operating on incompatible types (str vs int, list vs set, naive vs aware datetime).
Fix: Convert to a consistent type at the earliest point using the idiomatic conversion.

**P9 — Incorrect string/regex pattern**
Bug: A regex or string pattern does not match the intended input.
Fix: Correct the pattern. Test mentally against the failing input from the problem statement.

**P10 — Initialization order or missing setup**
Bug: A method is called before the object is fully initialized, or a required field is never set.
Fix: Move initialization earlier, or add the missing assignment. Check `__init__` for the
     correct location.

When none of these patterns match, describe your pattern explicitly in `fix_strategy`
so the Critic can evaluate whether a simpler pattern was available.

## Complexity escalation rule
Cycle 1: use the simplest matching pattern from the library. If the fix requires
more than 5 lines changed across all files, reconsider — a simpler pattern likely applies.
Retry cycles: you may escalate, but `fix_strategy` must state why cycle 1's simpler
pattern was insufficient.

## Hard rules
- Read relevant_files IN FULL before proposing.
- The downstream contract check in Step 3 is not optional — do it every time.
- Never propose version-detection branches if a single idiomatic call exists.
- Never propose a partial fix — if downstream lines break, fix them too.
- Never modify any files — proposal only.
- The JSON block must be the VERY LAST thing in your response.
- Never add error handling (try/except, if-checks on return values) at a call site
  to paper over a bug that lives inside the called function. Fix it there.
- Never catch a built-in exception type (RuntimeError, ValueError, Exception) unless
  you have verified there is exactly one raise statement in the called function body
  and it is the one you intend to catch.
- A fix proposal that contradicts any maintainer constraint recorded before Step 1
  is invalid, regardless of whether it solves the crash.
- If exploration_notes quotes a PR diff, your fix must either follow the same pattern
  or explicitly state in the rationale why a different pattern is more appropriate.
  Ignoring a quoted PR diff is not permitted.
"""


# ---------------------------------------------------------------------------
# Initial message builder
# ---------------------------------------------------------------------------

def build_initial_message(
    context: dict,
    localization_state: dict,
    previous_proposal: str | None = None,
    previous_patch: str | None = None,
    critic_output: str | None = None,
) -> str:
    message = (
        "Produce a solution proposal for the following bug.\n\n"
        f"<instance_id>{context['instance_id']}</instance_id>\n\n"
        f"<problem_statement>\n{context['problem_statement']}\n</problem_statement>\n\n"
        f"<hints>\n{context['hints_text']}\n</hints>\n\n"
        "<localization_report>\n"
        f"Relevant files: {json.dumps(localization_state['relevant_files'], indent=2)}\n"
        f"Relevant symbols: {json.dumps(localization_state['relevant_symbols'], indent=2)}\n"
        f"Bug analysis: {localization_state['bug_analysis']}\n"
        f"Exploration notes: {localization_state['exploration_notes']}\n"
        "</localization_report>\n\n"
        "Start by reading the relevant files, then produce your solution proposal.\n"
    )

    # On retry cycles, include previous attempt context
    if previous_proposal and previous_patch and critic_output:
        message += (
            "\n## Previous fix attempt failed — use this to refine your proposal\n\n"
            f"<previous_proposal>\n{previous_proposal}\n</previous_proposal>\n\n"
            f"<previous_patch>\n{previous_patch}\n</previous_patch>\n\n"
            f"<critic_output>\n{critic_output}\n</critic_output>\n\n"
            "Analyze what went wrong with the previous fix and propose a better approach.\n"
        )

    return message


# ---------------------------------------------------------------------------
# Output parsing
# ---------------------------------------------------------------------------

def parse_output(final_response: str) -> dict | None:
    try:
        start = final_response.rfind("```json")
        end   = final_response.rfind("```", start + 1)
        if start == -1 or end == -1:
            return None
        json_str = final_response[start + 7:end].strip()
        parsed = json.loads(json_str)
        required = {"files_to_modify", "proposed_changes", "fix_strategy"}
        if not required.issubset(parsed.keys()):
            return None
        return parsed
    except (json.JSONDecodeError, ValueError):
        return None


# ---------------------------------------------------------------------------
# State persistence
# ---------------------------------------------------------------------------

def save_state(instance_id: str, state: dict, cycle: int) -> Path:
    state_path = STATE_DIR / instance_id / f"architect_cycle{cycle}.json"
    state_path.parent.mkdir(parents=True, exist_ok=True)
    state_path.write_text(json.dumps(state, indent=2, cls=_SafeEncoder))
    get_logger().info(f"[architect] state saved → {state_path}")
    return state_path


def load_state(instance_id: str, cycle: int) -> dict | None:
    state_path = STATE_DIR / instance_id / f"architect_cycle{cycle}.json"
    if state_path.exists():
        try:
            text = state_path.read_text().strip()
            if not text:
                return None
            return json.loads(text)
        except (json.JSONDecodeError, ValueError):
            return None
    return None


class _SafeEncoder(json.JSONEncoder):
    """JSON encoder that converts any non-serializable object to its string repr."""
    def default(self, obj):
        if hasattr(obj, "__dict__"):
            return obj.__dict__
        return str(obj)


def serialize_messages(messages: list) -> list:
    """
    Convert Anthropic SDK message history to a JSON-safe list.
    Uses _SafeEncoder to handle any SDK-specific types (TextBlock,
    ToolUseBlock, DirectCaller, etc.) that are not natively serializable.
    """
    raw = json.dumps(
        [{"role": m["role"], "content": m["content"]} for m in messages],
        cls=_SafeEncoder,
    )
    return json.loads(raw)


# ---------------------------------------------------------------------------
# Main entry point
# ---------------------------------------------------------------------------

def run_architect(
    context: dict,
    container,
    localization_state: dict,
    cycle: int = 1,
    previous_proposal: str | None = None,
    previous_patch: str | None = None,
    critic_output: str | None = None,
) -> dict:
    """
    Run the Architect Agent for a SWE-bench instance.

    Args:
        context            : agent context bundle
        container          : running Docker container
        localization_state : output from run_localization()
        cycle              : fix/verify cycle number (1-based)
        previous_proposal  : Architect's previous proposal text (retry only)
        previous_patch     : Editor's previous patch (retry only)
        critic_output      : Critic's test output (retry only)

    Returns:
        State dict with status, files_to_modify, proposed_changes, fix_strategy
    """
    instance_id = context["instance_id"]

    existing = load_state(instance_id, cycle)
    if existing and existing.get("status") == "success":
        get_logger().info(f"[architect] found existing state for {instance_id} cycle {cycle} — skipping.")
        return existing

    bash_tool                  = make_bash_tool(container)
    get_classes_tool           = make_get_classes_and_methods_tool(container)
    extract_method_tool        = make_extract_method_tool(container)
    find_similar_api_calls_tool = make_find_similar_api_calls_tool(container)
    last_result: AgentResult | None = None

    for attempt in range(1, MAX_RETRIES + 1):
        get_logger().info(f"[architect] attempt {attempt}/{MAX_RETRIES} for {instance_id} cycle {cycle}")

        initial = build_initial_message(
            context, localization_state,
            previous_proposal, previous_patch, critic_output
        )

        last_result = run_agent_loop_auto(
            instance_id=instance_id,
            system_prompt=SYSTEM_PROMPT,
            initial_message=initial,
            tools=[bash_tool, get_classes_tool, extract_method_tool, find_similar_api_calls_tool],
            model=MODEL,
            max_iterations=MAX_ITER,
            verbose=True,
        )

        if not last_result.success:
            get_logger().info(f"[architect] attempt {attempt} hit MAX_ITERATIONS.")
            continue

        parsed = parse_output(last_result.final_response)
        if parsed is None:
            get_logger().info(f"[architect] attempt {attempt} did not produce valid JSON.")
            continue

        state = {
            "instance_id":      instance_id,
            "status":           "success",
            "cycle":            cycle,
            "attempt":          attempt,
            "files_to_modify":  parsed["files_to_modify"],
            "proposed_changes": parsed["proposed_changes"],
            "fix_strategy":     parsed["fix_strategy"],
            "full_response":    last_result.final_response,
            "messages":         serialize_messages(last_result.messages),
            "iterations":       last_result.iterations,
            "timestamp":        time.strftime("%Y-%m-%dT%H:%M:%SZ", time.gmtime()),
        }
        save_state(instance_id, state, cycle)
        return state

    get_logger().info(f"[architect] all {MAX_RETRIES} attempts failed for {instance_id} cycle {cycle}.")
    state = {
        "instance_id":      instance_id,
        "status":           "failed",
        "cycle":            cycle,
        "attempts":         MAX_RETRIES,
        "files_to_modify":  [],
        "proposed_changes": [],
        "fix_strategy":     "",
        "full_response":    last_result.final_response if last_result else "",
        "messages":         serialize_messages(last_result.messages) if last_result else [],
        "iterations":       last_result.iterations if last_result else 0,
        "timestamp":        time.strftime("%Y-%m-%dT%H:%M:%SZ", time.gmtime()),
    }
    save_state(instance_id, state, cycle)
    return state
